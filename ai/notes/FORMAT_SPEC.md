# 文章整理格式规范

> 用于规范从微信公众号/技术博客等来源整理的学习笔记

---

## 📁 文件存储规范

### 1. 存储路径
```
notes/
├── AI_深度学习/
│   ├── DETR/
│   ├── NLP/
│   ├── Transformer/
│   ├── MLOps_机器学习工程/
│   ├── 多模态_内容理解/
│   └── 自监督学习_表示学习/
├── 后端开发/
├── 前端开发/
├── 算法与数据结构/
├── 系统设计/
└── DevOps/
```

### 2. 文件名格式
```
文章标题.md

示例：
- 美团搜索中NER技术的探索与实践.md
- Transformer位置编码综述.md
- DETR及其变体综述.md
- BERT在搜索排序中的实践.md
- 知识蒸馏技术详解.md
```

**命名规则**：
- ✅ 主标题使用**中文**（便于识别和搜索）
- ✅ **技术关键词保留英文**（Transformer、BERT、NER、DETR、CNN、RNN、LLM等）
- ❌ 不使用日期前缀
- ❌ 不保留"Update"后缀的版本，只保留完整的最终版本
- 如果内容有更新，直接在原文档中添加新章节，而非创建新文件

**技术关键词示例**：
| 保留英文 | 说明 |
|---------|------|
| Transformer | 模型架构名称 |
| BERT/GPT/RoBERTa | 预训练模型名称 |
| NER | Named Entity Recognition |
| DETR | DEtection TRansformer |
| CNN/RNN/LSTM/GRU | 神经网络结构 |
| LLM | Large Language Model |
| CV/NLP | 计算机视觉/自然语言处理 |
| API/SDK | 技术接口/开发工具包 |

### 3. 禁止的操作
- ❌ 不要保存在根目录
- ❌ 不要使用中文日期（如：2026年2月14日）
- ❌ 不要省略日期前缀

## ✅ 整理检查清单

在保存文档前，确认以下事项：

- [ ] 文件名使用中文标题（不含日期前缀）
- [ ] 文件保存在正确的子目录下
- [ ] 元信息完整（原文链接、作者、发布时间等）
- [ ] 包含"一句话总结"部分
- [ ] 包含"核心要点"部分
- [ ] 正文结构清晰，使用Markdown标题层级
- [ ] 添加了适当的标签
- [ ] 更新了README.md目录
- [ ] 删除了临时文件或旧版本

## 📌 快速指令模板

### 整理单篇文章
```
阅读以下链接的文章，按照格式规范整理成笔记：
URL: {文章链接}

要求：
1. 提取元信息（标题、作者、发布时间）
2. 生成一句话总结
3. 提炼3-5个核心要点
4. 详细整理正文内容
5. 添加个人思考
6. 保存到 notes/AI_深度学习/{对应子目录}/{标题}.md
7. 更新 notes/README.md
```

### 批量整理多篇文章
```
整理以下 {N} 篇文章：
{文章1链接}
{文章2链接}
...

要求：
1. 自动分类到对应子目录
2. 按照格式规范生成每篇笔记
3. 更新README目录
4. 返回处理摘要
```
